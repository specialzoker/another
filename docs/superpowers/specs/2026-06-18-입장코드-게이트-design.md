# 입장코드 게이트 설계 (Supabase 기반, 박스플롯 앱과 동일 방식)

## 목적
지원경향 분석 웹앱에 박스플롯 앱(https://specialzoker.github.io/ggjinhyubboxplot/)과 **동일한 입장 방식**을 적용한다. 같은 Supabase 프로젝트의 `validate-key` 엣지 함수로 학교 고유 키를 검증해, 통과해야 앱 화면이 보인다.

## 합의된 결정
- **소프트 게이트**: 화면(UI)만 가린다. 데이터(`web/data/*.js`)는 GitHub Pages 정적 파일이라 주소를 알면 여전히 받을 수 있다(데이터 자체 보호는 목표 아님).
- **관리자 로그인 포함**: 박스플롯과 동일하게 이메일+비밀번호 로그인 UI도 둔다(이 앱에선 추가 기능은 없고 입장 경로로만 동작).
- **입장 기억**: 한 번 통과하면 `localStorage`에 저장해 재방문 시 게이트를 건너뛴다. 로그아웃으로 해제.
- 같은 Supabase 프로젝트를 쓰므로 **박스플롯에서 발급한 학교 키가 그대로 통한다.**

## 재사용하는 박스플롯 입장 로직 (그대로 활용)
```
SUPABASE_URL = 'https://qctffhrkhuaquciinblj.supabase.co'
SUPABASE_KEY = 'sb_publishable_4FZ0r-MqFdCjN4a1tRvLIQ_UmJVx0mj'   // publishable(공개) 키
EDGE_FN_URL  = SUPABASE_URL + '/functions/v1/validate-key'
```
- `doLogin()`: 키를 `EDGE_FN_URL`에 POST `{key}` → `res.ok` 면 통과(반환된 `data.school`로 학교명 표시). 박스플롯은 `data.admissions`도 받지만 **이 앱은 무시**(정적 데이터 사용).
- `doAdminLogin()`: `SUPABASE_URL/auth/v1/token?grant_type=password` 로 이메일+비번 → `access_token` 받으면 통과(라벨 '관리자').
- `toggleAdminMode()`, `doResetPw()`(비번 재설정 링크 복구), `setBanner()`: 박스플롯과 동일하게 가져옴.

## 아키텍처
- 신규 `web/auth.js` (app.js보다 먼저 로드): 위 상수·게이트 로직 + 입장 성공 시 처리.
- `web/index.html`: 게이트 오버레이 `#gate`(키 입력 / 관리자 토글·이메일·비번 / 배너·에러 / 로그아웃) 추가. `<body class="locked">` 로 시작.
- `web/styles.css`: 게이트 스타일을 앱 톤(블루/네이비 카드)으로 추가. `body.locked header, body.locked main{display:none}`, `#gate{display:none} body.locked #gate{display:flex}`.
- `web/app.js`: 현재 `document.addEventListener('DOMContentLoaded', init)` 자동 실행을 제거하고 `window.App.start = init` 로 노출. (입장 전 앱 미구동.)

## 입장 흐름 (데이터 흐름)
1. 페이지 로드 → `auth.js`가 `localStorage['gate_entry']` 확인.
   - 있으면 → `body.locked` 해제 + `App.start()` (게이트 스킵).
   - 없으면 → 게이트 표시(`body.locked` 유지).
2. 학교 키 입력 → `doLogin` → validate-key 통과 → `localStorage['gate_entry']={name, ts}` 저장 → unlock + `App.start()`.
3. 관리자 토글 → `doAdminLogin` → auth 토큰 통과 → 동일 저장(name='관리자') → unlock + `App.start()`.
4. 로그아웃 버튼 → `localStorage` 삭제 → `body.locked` 재설정 → 게이트 표시(페이지 리로드).

## 에러 처리
- 빈 키/빈 자격 → 안내 문구.
- validate-key 실패(`!res.ok`) → `data.error || '유효하지 않은 키입니다.'` 표시.
- 관리자 인증 실패 → '이메일 또는 비밀번호가 올바르지 않습니다.'
- 네트워크 오류 → '연결 오류: ...' (Supabase 미도달 시).

## 보안/한계 (명시)
- 클라이언트 게이트 + `localStorage` 플래그는 우회 가능(소프트 게이트). 데이터 파일은 공개 상태 유지.
- `SUPABASE_KEY`는 publishable 공개 키(박스플롯 클라이언트에 이미 공개됨) — 비밀 아님, 재사용 안전.

## 테스트
- 게이트 표시(잠금 상태에서 헤더·본문 숨김, 게이트만 보임).
- 빈 키 → 에러, 잘못된 키 → validate-key 에러 표시(네트워크 닿을 때).
- 입장 성공 시(성공 경로를 직접/모의 호출로) `body.locked` 해제·`App.start()` 실행·검색 동작.
- `localStorage` 기억: 입장표시 있으면 재로드 시 게이트 스킵.
- 로그아웃 → 표시 삭제 → 게이트 재표시.
- 실제 유효키 왕복은 사용자 키로 최종 확인(제작자가 보유). 빌드/검증 단계에서 Supabase 도달 여부 확인.

## 범위 밖
- 데이터의 실제 비공개화(Supabase 이전) — 채택 안 함(소프트 게이트로 합의).
- 박스플롯 고유의 학교별 데이터 로딩·관리자 키 관리 기능 — 이 앱엔 불필요.
