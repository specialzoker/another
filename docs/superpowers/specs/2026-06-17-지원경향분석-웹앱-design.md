# 지원경향분석 웹앱 설계 (2026 교차지원 경향)

## 목적
작년 입시에서 **특정 대학·전형(또는 학과)에 지원한 학생들이 또 어디에 지원했고 어디에 합격했는지**(교차지원 경향)를 선생님·학생이 쉽게 조회할 수 있는 정적 웹앱.

- 사용 대상: 진학 상담 선생님 + 학생/학부모 (공용 도구)
- 핵심 가치: 설치·서버 없이 파일만 공유하면 어디서나 열려 동작, 오프라인 가능

## 원본 데이터
파일: `2026 지원경향분석_전형_학과.xlsx` (시트 4개)

| 시트 | 단위 | 행수 | 비고 |
|---|---|---|---|
| 전형 | 대학·전형 | 1,901 | 1~10지망 |
| 전형_인문 | 대학·전형(인문) | 962 | 1~10지망 |
| 전형_자연 | 대학·전형(자연) | 1,275 | 1~10지망 |
| 학과 | 대학·전형·모집단위 | 13,496 | 1~9지망, 모집단위 칼럼 추가 |

**열 구성**: 지역 / 대학명 / 전형유형 / 전형명 / (모집단위) / 사례수 / 최고·평균·중간·최저(내신등급) / 1지망…N지망

**셀 의미**
- `사례수` `94 [33]` = 94명 지원, 그중 **33명이 이 전형에 합격**
- 각 지망 칸 = 같이 지원한 다른 대학·전형(학과 시트는 학과명 포함)을 지원자 많은 순으로 나열. 칸 내부는 여러 줄 텍스트로 `대학명 / (전형명) / N[M]` 형태이며 `N[M]` = N명 같이 지원, M명 거기 합격.
- 내신등급 4개 = 해당 전형 합격선 분포(최고~최저)

## 아키텍처
```

├─ data/                      # 생성물(빌드 산출)
│   ├─ index.json             # 검색용 목록(시트별 대학·전형·학과 + id)
│   ├─ jeonhyeong.json
│   ├─ jeonhyeong_inmun.json
│   ├─ jeonhyeong_jayeon.json
│   ├─ hakgwa.json
│   └─ build_report.json      # 행 수·매칭률·스킵된 셀 리포트
└─ web/                       # 정적 사이트(서버 불필요)
    ├─ index.html
    ├─ app.js
```

- `web/` 폴더만 통째로 공유 → `index.html` 더블클릭 실행. (브라우저 `file://`에서 `fetch` 제약이 있으면 데이터를 `data.js`의 전역 변수로 임베드하는 방식으로 폴백.)
- 시트를 고를 때 **해당 시트 JSON만 지연 로드**해 초기 로딩을 가볍게(학과 데이터가 큼).

## 데이터 모델 (정규화된 한 행)
```jsonc
{
  "id": "전형:서울대:학생부종합:지역균형",   // 시트:대학:유형:전형(:모집단위)
  "sheet": "전형",
  "region": "서울",
  "university": "서울대",
  "type": "학생부종합",
  "name": "지역균형",
  "unit": null,                              // 학과 시트만 모집단위
  "count": { "applied": 94, "accepted": 33 },
  "rate": 0.351,                             // accepted/applied (표시용, 0분모 보호)
  "grades": { "best": 1.12, "avg": 1.24, "median": 1.22, "worst": 1.45 },
  "preferences": [
    {
      "rank": 1,
      "university": "고려대",
      "label": "학교추천",                   // 전형명(학과 시트는 "전형 · 학과")
      "applied": 60,
      "accepted": 44,
      "rate": 0.733,
      "matchedId": "전형:고려대:..."          // 역방향 연결, 실패 시 null
    }
  ]
}
```
`index.json`은 `[{id, sheet, region, university, type, name, unit, label}]` 형태의 검색용 경량 목록.

## 핵심 로직
### 지망 칸 파싱
- 칸 텍스트를 줄 단위로 분리(원본의 `_x000D_`/개행 정리).
- **마지막 줄**이 `^\s*(\d+)\s*\[\s*(\d+)\s*\]\s*$` 매칭 → `applied`, `accepted`.
- 그 앞 줄들 = 라벨. 첫 줄=대학명, 나머지=전형명/학과명(학과 시트는 전형명+학과명 두 줄일 수 있음 → " · "로 합쳐 표시).
- 형식 불일치 셀은 건너뛰고 `build_report.json`에 기록(앱 동작에는 영향 없음).

### 역방향 매칭 (네트워크·양방향 비교용)
- 키 = `정규화(대학명) + "|" + 정규화(전형명)`.
- 정규화: 양끝 공백·괄호`()`·대괄호`[]` 내용·캠퍼스 접미(광주/경산/칠암/아산 등)·"전형" 접미어·공백 제거.
- 같은 시트 내 원본 행 인덱스와 best-effort 매칭. 실패 시 `matchedId=null`.
- 빌드 시 **시트별 매칭률**을 리포트로 출력. (완벽한 100% 매칭은 표기 차이로 보장 불가 — 의도된 한계.)

## 화면 구성 (UX)
디자인 톤: 사용자의 기존 상담 웹앱과 동일 계열.
- Pretendard / Noto Sans KR, 옅은 블루 그라데이션 배경(`#f8fbff→#eef4fb`)
- 흰색 라운드 카드(반경 20px, 1px `#dbe4ef`, 은은한 그림자)
- 브랜드 블루 `#1d4ed8` / 네이비 `#072b55`, 포인트 주황 `#f97316`
- 알약형 탭·배지(good/ok/warn/bad 색), 상단 KPI 타일, sticky 헤더 테이블, 블루 그라데이션 primary 버튼

**(A) 검색 화면**
- 상단 시트 탭: 전형 / 인문 / 자연 / 학과
- 검색창(대학·전형·학과 자동완성) + 필터(지역, 전형유형)
- 결과 후보 리스트(클릭 시 순위표로)

**(B) 순위표 화면 (기본)**
- 선택 전형 **요약 카드**: 사례수 `N[M]`, 합격률(배지), 내신 최고·평균·중간·최저 KPI 타일
- **같이 지원한 전형 순위표**: 지망순위 / 대학·전형(·학과) / 지원수 / 합격수 / 합격률(막대그래프 셀)
- 행 **더블클릭 → 드릴인**

**(C) 드릴인**
- ① 교차지원 네트워크/흐름도: 중심=선택 전형, 엣지 굵기=지원수, 노드 클릭으로 탐색
- ② 양방향 비교: A→B 지원 비중 vs B→A 지원 비중을 나란히. `matchedId=null`이면 "역방향 데이터 없음" 배지.

## 에러 처리
- 검색 결과 없음 → 안내 문구.
- 역방향 매칭 실패 → "역방향 데이터 없음" 배지(앱은 정상).
- 파싱 형식 불일치 셀 → 스킵 + 리포트 기록.
- `file://`에서 `fetch` 차단 시 → 임베드(`data.js`) 폴백으로 항상 열리게.

## 테스트
- **파서 단위 테스트**(`build/test_parse.py`): 알려진 셀 입력 → 기대 `{applied, accepted, label}`; `N[M]` 추출; 학과 시트 다중 줄 라벨; 형식 불일치 스킵.
- **정규화·매칭 테스트**: 알려진 쌍(예: 지망 "고려대 학교추천" ↔ 원본 "고려대 학생부교과 학교추천")이 매칭되는지.
- **빌드 검증**: 시트별 행 수가 원본과 일치, 매칭률 리포트 생성.

## 범위 밖 (YAGNI)
- 연도 간 비교, 로그인/저장, 합격 예측 모델, 데이터 편집 기능 — 이번 범위 아님.
