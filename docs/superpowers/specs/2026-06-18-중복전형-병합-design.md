# 중복 전형 병합 설계 (전형 목록 정리)

## 배경 / 문제
검색 목록에서 같은 전형이 여러 번 뜬다. 원인: 원본 엑셀이 같은 전형을 **모집단위/계열별로 분리 기록**했는데 전형 시트엔 모집단위 칸이 없어 `지역·대학·전형유형·전형명`이 동일한 행이 2~5개씩 보임. 빌드에서 이들에 `#2`,`#3` 접미 id를 붙여 개별 도달은 가능하나, 목록이 지저분하다.

규모(현재): 전형 시트 1900행 중 **163개 그룹(349행)이 표시상 중복**(그룹 크기 2~5). 인문/자연도 유사, 학과는 모집단위가 있어 거의 없음.

## 목표
표시상 동일한 행들을 **하나로 완전 병합**해 목록을 정리한다. 캠퍼스가 다른 경우(예: `경희대` vs `경희대(국)`)는 대학명이 달라 자동으로 분리 유지된다.

## 결정 사항
- **완전 병합**: 한 전형 = 한 줄. 사례수 합산, 내신 범위, 같이 지원한 전형 합쳐 재집계.
- **캠퍼스 분리 유지**: 서울캠/국제캠 등은 별개 입시이므로 섞지 않음.

## 병합 단위 (그룹 키)
`base-id` = 레코드 `id`에서 끝의 `#숫자` 접미를 제거한 값. 이는 `시트키:대학:전형유형:전형명(:모집단위)`와 동일하며, 곧 "표시상 동일 식별자". region은 대학명에 종속(대학→지역 1:1)이라 키에 불필요.

## 병합 규칙 (그룹 → 단일 레코드)
- `id`: base-id (접미 제거 → 그룹당 1개로 유일)
- `count.applied` = Σapplied, `count.accepted` = Σaccepted, `rate` = accepted/applied (0분모 보호)
- `grades.best` = min(best), `grades.worst` = max(worst), `grades.avg`·`grades.median` = **applied 가중평균**(None 세그먼트는 제외, 전부 None이면 None)
- `region/university/type/name/unit/sheet`: 그룹 공통값(첫 레코드 기준)
- `preferences`: 모든 세그먼트의 지망을 **정규화된 대상 키**(normalize_university(u)+"|"+normalize_name(label))로 묶어 `applied`·`accepted` 합산 → `rate` 재계산 → `applied` 내림차순 정렬 → `rank` 1..N 재부여. 표시용 `university`/`label`은 그룹 내 첫 등장값 사용. 병합 후 모든 대상 유지(상한 없음).

## 빌드 순서 (build/)
1. `parse_sheet` → records (현행, `#n` 접미 포함)
2. **merge_records(records)** → base-id로 그룹화해 위 규칙으로 병합한 새 records 리스트
3. `build_match_index(merged)` → 매칭 인덱스
4. 각 병합 레코드의 합쳐진 preference에 `matchedId`=find_match(...), `rate` 설정 / 레코드 `rate` 설정
5. `web/data/*.js` + `index.js` + `report.json` 출력 (행수는 병합 후 기준)

프런트엔드(`web/`)는 **레코드 구조가 동일**하므로 수정 없음. 검색 목록·순위표·드릴인·양방향 모두 그대로 동작.

## 구현 위치
- 신규: `build/parse.py`에 `merge_records(records)` (순수 함수)
- 수정: `build/build.py`에서 parse 직후 merge 호출, 이후 매칭/출력
- 테스트: `build/tests/test_merge.py`

## 테스트
- **단위(merge_records)**: 같은 base-id 3행 입력 → 1행, applied/accepted 합산, best=min·worst=max, avg=가중평균, 지망이 같은 대상끼리 합산·재순위. 서로 다른 base-id는 안 묶임(캠퍼스 분리).
- **빌드 검증**: 병합 후 모든 시트 id 유일, 행수 감소(전형 1900→~1714), 매칭률 리포트.
- **브라우저 실측**: `경희대`·`경상국립대 일반` 검색 시 중복 사라지고, 합쳐진 사례수·내신 범위·교차지원 목록이 맞는지 확인.

## 범위 밖
- 캠퍼스 통합, 묶되-펼치기 UI, 런타임 병합 — 채택 안 함.
